Étape 3 : Configuration IP statique avec Netplan

Important :
Ubuntu Server 22 utilise Netplan pour la configuration réseau (et non plus /etc/network/interfaces). La configuration se fait dans des fichiers YAML.

Une fois Ubuntu installé, j'ai configuré l'adresse IP statique attribuée : 100.115.29.182/23

Édition du fichier Netplan

# 1. Éditer le fichier de configuration réseau
sudo nano /etc/netplan/00-installer-config.yaml
# Contenu du fichier YAML
network:
  ethernets:
    ens160:
      addresses: [100.115.29.182/23]
      routes:
        - to: default
          via: 100.115.29.254
      nameservers:
        addresses: [100.115.29.254, 100.115.28.41, 8.8.8.8]
      dhcp4: false
      dhcp6: false
  version: 2
Explication de la configuration :
  • ens160 : nom de l'interface réseau (peut varier)
  • dhcp4(6): false : désactive DHCP (on veut du statique)
  • addresses: [100.115.29.182/23] : adresse IP + masque (/23 = 255.255.254.0)
  • via: 100.115.29.254 : passerelle par défaut
  • addresses: [...] : serveurs DNS du lycée et de Google

Configuration de l'adresse IP sur la VM US22

Application de la configuration

# 2. Appliquer la configuration réseau
sudo netplan apply

# 3. Tester la connexion
ping 8.8.8.8
Résultat :
L'adresse IP 100.115.29.182 est bien configurée et la VM peut accéder à Internet !

Étape 5 : Installation d'Odoo 16

J'ai suivi la procédure officielle d'installation d'Odoo 16 sur Ubuntu Server.

Installation des dépendances

# Installation de PostgreSQL (base de données)
sudo apt install postgresql postgresql-client -y

# Installation de Python 3 et pip
sudo apt install python3 python3-pip python3-dev python3-venv -y

# Installation des bibliothèques système nécessaires
sudo apt install libxml2-dev libxslt1-dev libldap2-dev libsasl2-dev \
    libtiff5-dev libjpeg8-dev libopenjp2-7-dev zlib1g-dev \
    libfreetype6-dev liblcms2-dev libwebp-dev -y

# Installation de wkhtmltopdf (pour générer des PDF)
sudo apt install wkhtmltopdf -y

Création de l'utilisateur Odoo

# Créer un utilisateur système pour Odoo
sudo adduser --system --home=/opt/odoo --group odoo

Configuration de PostgreSQL

# Créer un utilisateur PostgreSQL pour Odoo
sudo su - postgres -c "createuser -s odoo"

Téléchargement d'Odoo 16

# Se placer dans /opt
cd /opt

# Cloner le dépôt Odoo 16
sudo git clone https://www.github.com/odoo/odoo --depth 1 --branch 16.0 /opt/odoo16

# Donner les droits à l'utilisateur odoo
sudo chown -R odoo:odoo /opt/odoo16

Installation des dépendances Python

# Installer les requirements Python
sudo pip3 install -r /opt/odoo16/requirements.txt

Création du fichier de configuration

# Créer le fichier de configuration Odoo
sudo nano /etc/odoo16.conf
# Contenu du fichier de configuration
[options]
admin_passwd = stssio
db_host = localhost
db_port = 5432
db_user = odoo
db_password = False
addons_path = /opt/odoo16/addons
logfile = /var/log/odoo16/odoo.log

Démarrage d'Odoo

# Créer le dossier de logs
sudo mkdir /var/log/odoo16
sudo chown odoo:odoo /var/log/odoo16

# Démarrer Odoo manuellement pour test
sudo su - odoo -s /bin/bash -c "/opt/odoo16/odoo-bin -c /etc/odoo16.conf"
Odoo démarré !
Odoo est maintenant accessible sur http://100.115.29.182:8069

Problème détecté :
Par défaut, Odoo n'a qu'un seul champ "Nom" qui contient le nom complet. Il n'y a pas de séparation Nom / Prénom !

→ Ce problème sera résolu dans la Partie 2 avec l'installation d'un addon.

Étape 9 : Test d'import du fichier CSV exemple

J'ai testé l'import du fichier CSV exemple fourni (clients_particuliers_exemple.csv) :

# Structure du fichier CSV exemple
Titre;Nom;Prénom;Email;Téléphone;Mobile;Code postal;Ville;Rue;Est une entreprise
Madame;Perrin;Guillemette;guillemette.perrin@email.fr;0101234567;0701234567;45100;Orléans;10 Rue de la Marine;FAUX

Procédure d'import

  1. Menu Vente → Clients
  2. Clic sur Favoris → Importer
  3. Sélection du fichier CSV
  4. Mapping des colonnes CSV vers les champs Odoo
  5. Test de l'import
Problème constaté :
Les colonnes Nom et Prénom du CSV ne peuvent pas être mappées correctement car Odoo n'a qu'un seul champ "Nom" (nom complet).

Conséquence : Impossible d'importer les données avec Nom et Prénom séparés sans addon !
