Création VM & Installation Debian

Création de la machine virtuelle Zabbix sur Proxmox et installation graphique complète de Debian 13.3.0

Accueil › Stage 2 › Missions › Mission 2 › Partie 3.1

Introduction à l'installation de Zabbix

Une fois l'installation Proxmox terminée, je vais donc passer à la seconde partie de ma mission qui est de créer une machine virtuelle Zabbix, qui aura les mêmes fonctionnalités et la même utilité que Spectrum (voir DT_RealisationSchemaReseau).

Note importante : Le seul nœud disponible est S36RESPVE01 car il s'agit du serveur Proxmox. Les VM ID sont choisis à la chaîne, c'est-à-dire que les VM ID commencent à 100 et qu'une machine virtuelle avait déjà été créée, celle que je vais créer ici est la deuxième donc la VM ID de cette machine virtuelle sera de réaliser des tests Zabbix afin de ne pas endommager la machine Zabbix de production.

La machine virtuelle sera donc créée sous l'environnement Proxmox, je clique donc sur « Créer une VM » dans l'interface web Proxmox.

Configuration générale de la VM

Identification de la machine virtuelle

Paramètres généraux : Nœud, VM ID et nom de la machine virtuelle

Nœud : S36RESPVE01

VM ID : 101

Nom : ZABBIX-SANDBOX (car l'utilité de cette machine virtuelle est de réaliser des tests)

Sélection de l'image ISO

Choix de l'image ISO Debian 13.3.0 depuis le stockage Images_ISO

Afin de choisir l'image ISO, sélection du dossier Images_ISO où toutes les images ISO sont stockées. L'image ISO choisie est donc la debian 13.3.0 qui correspond à la dernière version de Debian.

Configuration système et matérielle

Agent QEMU

Ensuite, activation de l'agent QEMU qui permet de multiples améliorations pratiques :

  • Arrêts et redémarrages propres (sans l'agent, Proxmox agit comme s'il coupait directement l'alimentation)
  • Meilleure gestion des snapshots (moins de corruption des fichiers)
  • Il a bien d'autres fonctionnalités qui sont très pratiques pour l'entretien et le bon fonctionnement de la machine

Activation de l'agent QEMU et configuration par défaut

Sélection du disque DATA1 (Array B, 150Go)

Stockage

Sélection du disque DATA1 qui correspond au Array B créé dans l'installation du serveur Proxmox ainsi que 150go, car c'est seulement l'espace dont j'ai besoin pour la machine virtuelle.

Processeur et mémoire

Sélection de 1 processeur et 2 cœurs

Allocation de 16 Go de RAM (16368 MB)

Processeur : Sélection de 1 processeur et 2 cœurs pour faire fonctionner la machine.

Mémoire : Sélection d'environ 16 Go de RAM pour faire fonctionner la machine.

Configuration réseau

Bridge et VLAN

Configuration du pont Trunk avec VLAN correspondant au Bridge Aggregation

Le pont choisi est Trunk car il s'agit du Bridge Aggregation effectué lors de la configuration des interfaces nic0 sur le serveur Proxmox.

Choix donc du Vlan correspondant au Bridge Aggregation réalisé plus tôt.

Récapitulatif de création

Fin de la création de la machine virtuelle - Résumé de toutes les informations

Récapitulatif de la configuration :

• Agent : 1 (QEMU activé)

• Cores : 2

• CPU : x86-64-v2-AES

• ide2 : Images_ISO:iso/debian-13.3.0-amd64-netinst.iso,media=cdrom

• Memory : 16368

• Name : ZABBIX-SANDBOX

• net0 : virtio,bridge=Trunk,tag=1,firewall=1 (Pont sur Trunk avec vlan correspondant)

• Nodename : S36RESPVE01

• scsi0 : DATA1:150,iothread=on (Disque dur précédent 150Go)

• vmid : 101 (ID de la deuxième machine virtuelle créée sur Proxmox)

Installation graphique de Debian 13.3.0

Démarrage de la VM. J'arrive donc sous un menu d'installation pour Debian et je lance l'installation graphique.

Menu de démarrage de l'installation Debian - Sélection de "Graphical install"

Langue et localisation

Le choix de la langue est donc le français

Choix d'une adresse IP disponible dans le vlan attribué

Choix d'une adresse IP disponible dans le vlan attribué (masque de sous réseau : 255.255.252.0 /22) puis de la passerelle par défaut correspondante à ce réseau.

Configuration DNS et hostname

Choix du DNS

Choix du nom de la machine (identique au nom choisi lors de la création)

Indication du domaine de l'AD de la SNCF.

Mot de passe administrateur

Indication du mot de passe voulu

Création du compte utilisateur

Création du compte utilisateur nommé "administrateur" (qui sera également son identifiant)

Partitionnement du disque

Configuration du partitionnement

Choix d'utiliser un disque entier et sélection du disque de 150Gio créé plus tôt

J'indique donc que je termine donc le partitionnement et que j'applique les changements, puis j'indique qu'il faut bien appliquer les changements sur les disques concernés.

Schéma de partitionnement :
Tout dans une seule partition (recommandé pour les débutants)

Configuration des paquets et logiciels

Miroir Debian

On ne souhaite pas analyser d'autres supports d'installation

Configuration de l'outil de gestion des paquets

J'indique que l'on veut configurer l'outil de gestion des paquets saisir manuellement le miroir car nous allons rentrer les informations manuellement et non des hôtes miroir propre à un pays.

Configuration du miroir :
J'indique donc le nom d'hôte du port miroir de l'archive Debian ainsi que son répertoire. Pour cela je dois me rendre directement sur le site repos.it.sncf.fr (nom hôte du miroir), l'arrive sur une page avec des dossiers, je parcoure le chemin suivant : /os/debian/mirror/deb.debian.org/debian/ et c'est le chemin qui correspond au répertoire miroir.

Participation aux statistiques

On ne souhaite pas que la machine partage les paquets que l'on utilise pour une étude de statistiques

Sélection des logiciels

Les logiciels qui seront installés sont le serveur SSH et les utilitaires usuels du système

Installation du programme de démarrage appelé GRUB. Ce programme permet de démarrer la machine virtuelle directement sur l'OS Debian qui lui a été installé. Dans le cas où GRUB n'est pas installé, la machine virtuelle n'est pas capable de démarrer sur un OS et va donc seulement redémarrer depuis le début de l'installation Debian.

Choix du disque dur principal (150Go) pour l'installation du programme GRUB

Installation terminée

L'installation graphique de Debian 13.3.0 est terminée, redémarrage de la machine virtuelle
